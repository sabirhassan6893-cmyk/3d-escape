# 3D Escape

A playful browser-based interaction experiment where the goal is simple: click the **Escape** key.

The catch?  
The Escape key keeps escaping in 3D space.

## Why I built this

I built this as one of my first AI-assisted coding experiments to move beyond static design screens and understand how interaction, motion, timing, and feedback actually behave in the browser.

As a Product Designer, I've worked primarily in Figma and product workflows, and I'm actively exploring ways to get closer to implementation. This project was a small but fun way to experiment with interaction design through code.

The idea was intentionally silly, but the learning was real.

## What it explores

- Playful interaction design
- Cursor-based movement
- 3D-like motion behaviour
- Micro-interactions and user feedback
- AI-assisted prototyping
- Moving from design idea to working browser experience

## What I learned

This project helped me understand that even a simple interaction needs a lot of product/design decisions:

- How fast should the object move?
- When does an interaction become fun vs frustrating?
- How much randomness should be added?
- How should the user get feedback?
- What makes a tiny browser experiment feel alive?

It also helped me get more comfortable using AI as a coding partner, not to replace design thinking, but to quickly test ideas and learn implementation patterns.
